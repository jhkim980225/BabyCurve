# 태아 성장 백분위 계산기 — 설계 문서

- 작성일: 2026-06-20
- 상태: 승인됨 (구현 계획 작성 단계로 진행)

## 1. 개요

임산부가 **임신 주수 + 태아 계측치**를 입력하면, 선택한 **성장 표준** 기준으로 **백분위**를 계산해 **색상 밴드 성장 곡선** 위에 점으로 마킹하여 직관적으로 보여주는 서비스.

- **타겟**: 해외 다국가 (영어권 + 동남아 + 동아시아 + 중남미 등)
- **배포**: 안드로이드 앱(Google Play) + SEO 웹사이트를 **하나의 코드베이스**로 동시 제공
- **장기 목표**: 2~3년 주기 데이터 업데이트, 부모 키 등 변수 추가 등 지속 고도화 가능한 구조

## 2. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | Next.js (App Router), `output: 'export'` | 정적 사이트 생성(SSG) → SEO + 앱 번들 공용 |
| 언어 | TypeScript | |
| 스타일 | Tailwind CSS | 모노 글래스 디자인 시스템 |
| 차트 | D3 스케일 + 커스텀 SVG/React 컴포넌트 | 밴드 폴리곤 + 50p 라인 + 마킹 점 정밀 제어 |
| 이미지 저장 | `html-to-image` | 결과 카드 → PNG |
| 공유 | Web Share API | 미지원 시 이미지 저장 폴백 |
| i18n | `next-intl` | 언어별 정적 라우트 + `hreflang` |
| 앱 패키징 | Capacitor → Android APK/AAB | 정적 빌드 번들, 오프라인 동작 |

## 3. 디자인

- **테마**: 모노 글래스(글래스모피즘 + 모노톤), **블루 라이트** 기본
- **다크모드**: 기기 시스템 설정에 따라 자동 전환 (블루 딥 글래스), 수동 토글 추가 가능
- **그래프 스타일**: 색상 밴드형 (정상 범위를 색 영역으로 채우고 50번째 라인 + 마킹 점)
- **첫 진입 언어 선택**: 최초 진입 시 9개 언어를 모국어 표기로 선택하는 화면 표시(글래스 카드 목록). 선택값은 로컬 저장하여 재진입 시 생략하고, 헤더에서 언제든 변경 가능. (웹은 브라우저 `Accept-Language`로 기본값 제안)
- **화면 흐름**: 언어 선택(최초 1회) → 한 화면 스크롤형 (입력 → Calculate → 하단에 결과 카드 전개)
- 절제된 그라데이션, 프로스티드 블러, 얇은 라이트 보더, 큰 백분위 숫자 타이포

## 4. 지원 언어 (9개 로케일)

영어(en), 스페인어-멕시코(es-MX), 일본어(ja), 중국어 번체-대만(zh-TW), 중국어 간체(zh-CN), 한국어(ko), 태국어(th), 베트남어(vi), 인도네시아어(id)

- 모든 UI 문자열은 i18n 메시지로 분리, 하드코딩 금지
- SEO: 언어별 정적 라우트 + `hreflang` + 메타/OG 태그

## 5. 기능 범위 (v1)

1. **백분위 계산기** — EFW/BPD/HC/AC/FL 계측치, 여러 표준 선택, 색상 밴드 곡선 + 마킹 점
2. **EFW 자동 계산** — BPD/HC/AC/FL 입력 시 Hadlock 공식으로 추정체중(EFW) 산출
3. **측정 기록 & 성장 추적** — 로컬 저장(브라우저/앱 스토리지), 같은 곡선에 여러 점으로 추세 표시
4. **예정일/임신주수 계산기** — 마지막 생리일(LMP) 또는 초음파 기준으로 출산예정일·현재 주수 계산
5. **주차별 태아 발달 정보** — 클라이언트 제공 콘텐츠(글/이미지)를 JSON으로 구조화하여 표시
6. **첫 진입 언어 선택** — 최초 진입 시 9개 언어 선택 화면(모국어 표기), 선택값 로컬 저장, 헤더에서 변경 가능

## 6. 데이터 모델 (확장성의 핵심)

관리자 페이지 없이 **JSON 파일 교체/추가**로 업데이트:

```
/data/standards/
  index.json            ← 표준 목록 메타 (id, name, source, supported metrics)
  who.json
  intergrowth-21st.json
  hadlock-us.json
/data/development/
  index.json            ← 주차별 발달 정보 (언어별)
```

성장 표준 파일 구조:

```json
{
  "id": "who",
  "name": "WHO Fetal Growth",
  "source": "...",
  "metrics": {
    "efw": {
      "unit": "g",
      "weeks": {
        "30": { "p3": 0, "p10": 0, "p50": 0, "p90": 0, "p97": 0 }
      }
    },
    "bpd": {},
    "hc": {},
    "ac": {},
    "fl": {}
  }
}
```

- 새 표준/연도 갱신 = JSON 파일 추가 후 재빌드·재배포. **코드는 데이터에 무관하게 동작**.
- 계측 항목, 백분위 라인 종류도 데이터가 결정 (하드코딩 금지).

## 7. 백분위 산출 로직 (`lib/percentile.ts`)

- 입력 주수가 데이터 포인트 사이면 **선형 보간**으로 해당 주수의 백분위 컷오프 계산
- 입력값을 컷오프와 비교, 두 컷오프 사이는 보간하여 **연속 백분위(%)** 산출 (예: 68%)
- 순수 함수로 분리 → 단위 테스트 용이
- EFW 자동 계산은 `lib/efw.ts`에 Hadlock 공식 별도 분리

## 8. 컴포넌트 / 모듈 경계

| 단위 | 책임 | 의존 |
|---|---|---|
| `InputForm` | 입력·검증 UI | i18n |
| `GrowthChart` | D3 밴드 + 마킹 점(들) 렌더 | percentile 결과 |
| `ResultCard` | 백분위 결과 + 해석 + 면책 + 저장/공유 | GrowthChart, html-to-image |
| `lib/percentile.ts` | 백분위 계산(보간) | 표준 데이터 |
| `lib/efw.ts` | Hadlock EFW 계산 | — |
| `lib/data.ts` | 표준/발달정보 JSON 로딩 | data 파일 |
| `lib/storage.ts` | 측정 기록 로컬 저장/조회 | — |
| `lib/i18n` | 다국어 메시지 | next-intl |

각 단위는 독립적으로 이해·테스트 가능해야 함.

## 9. 에러 / 검증

- 범위 밖 주수, 음수, 빈값 → 인라인 검증 메시지(다국어)
- 선택 표준이 해당 계측치 미지원 시 안내
- **의료 면책**: "참고용이며 의학적 진단이 아님"을 전 화면 및 결과 이미지에 명시 (Google Play 의료 가이드라인 준수)

## 10. SEO

- 메타/OG 태그, 구조화 데이터(JSON-LD)
- `sitemap.xml`, `robots.txt`

## 14. 의존성 / 리스크

- **클라이언트 제공물**: ① 성장 표준 데이터/논문(표준별 백분위 표) ② 주차별 발달 정보 콘텐츠(9개 언어 또는 영문)
- **리스크**: 발달 정보 9개 언어 콘텐츠 준비 일정/분량이 클라이언트 측 작업이라 마일스톤에 영향을 줄 수 있음 → 콘텐츠 미수령 시 해당 기능은 영문 우선/후순위 처리 가능하도록 설계
- **Play 심사**: 단순 웹뷰 거절 방지를 위해 Capacitor 번들(오프라인 동작) + 네이티브 기능(저장/공유) 포함, 의료 면책 명시
